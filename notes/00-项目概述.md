# 00 - SGLang 项目概述

## 什么是 SGLang?

SGLang (Structured Generation Language) 是一个快速、灵活的大语言模型服务框架。

## 核心特性

1. **高性能服务**:
   - RadixAttention 算法用于 KV 缓存管理
   - 高效的批处理和调度策略

2. **结构化生成**:
   - 支持约束解码
   - JSON schema 输出
   - 正则表达式约束

3. **易用的编程接口**:
   - 声明式的语言API
   - 支持多种模型和后端

4. **生产级特性**:
   - 分布式推理
   - 多模态支持
   - LoRA 适配器

## 与其他框架对比

| 特性 | SGLang | vLLM | TGI |
|------|--------|------|-----|
| RadixAttention | ✅ | ❌ | ❌ |
| 结构化输出 | ✅ | 有限 | 有限 |
| 编程接口 | 高级API | 低级API | REST |
| 多模态 | ✅ | ✅ | ✅ |

## 技术栈

- **语言**: Python (前端), C++/CUDA (内核)
- **深度学习框架**: PyTorch
- **推理优化**: CUDA, Triton, FlashAttention
- **服务框架**: FastAPI, gRPC

## 核心概念

### 1. RadixAttention
- KV缓存的前缀树管理
- 自动共享公共前缀
- 减少内存使用和计算

### 2. Runtime (SRT)
- 核心运行时系统
- 负责请求调度、模型执行、内存管理

### 3. Language API
- 高层编程接口
- 声明式定义生成流程

## 源码结构概览

```
sglang/
├── srt/                    # Runtime 核心
│   ├── managers/           # 调度管理器
│   ├── model_executor/     # 模型执行
│   ├── mem_cache/          # RadixAttention缓存
│   ├── sampling/           # 采样策略
│   └── constrained/        # 约束解码
├── lang/                   # 语言API
├── jit_kernel/             # JIT编译内核
└── multimodal_gen/         # 多模态生成
```
